Archive & Index

One catalogue for everything DRIP holds – searchable, indexed by pillar, type and source, and built to stay current. Every knowledge product is logged with where it came from and when it last changed, so the platform never goes dormant again.

01

How the index stays current

New knowledge enters the archive through four channels. Whatever the route, each item is catalogued the same way – given a pillar, a type, a source and a date – so the index updates the moment it is added.

How updating works
  1. Add an item here – it’s saved in your browser as a draft and appears in the index immediately, tagged Draft.
  2. When ready, Export updated catalogue – this downloads a refreshed drip-archive.js containing every entry plus your drafts.
  3. Commit that file to the site (replace the old one). The drafts become permanent catalogue entries for everyone.
  4. The same flow handles reference files, Regional Exchange contributions and knowledge-hub pulls – set the source channel accordingly.

Drafts live only in your browser until exported and committed, so nothing is lost on refresh – but they aren’t shared until the catalogue file is updated.
